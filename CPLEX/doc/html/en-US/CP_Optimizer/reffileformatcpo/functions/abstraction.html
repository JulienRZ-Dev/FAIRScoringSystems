abstraction

Returns a constraint that abstracts the values of one array as values in another array.

Syntax


constraint abstraction(intExprArray y,  intExprArray x, intArray values, int abstractValue)

Parameters

  • y: an array of abstracted integer expressions
  • x: an array of reference integer expressions
  • values: an array of integer values to be abstracted
  • abstractValue: an escape value

Description

This constraint abstracts expression array x into the expression array y. That is:

  • x[i] = v with v in values if and only if y[i] = v.
  • x[i] = v with v not in values if and only if y[i] = abstractValue.

The abstract value (specified by abstractValue) must not be in the domain of x[i].

This constraint maintains a many-to-one mapping that makes it possible to define constraints that impinge only on a particular set of values.

Example


x1 = intVar(3);
x2 = intVar(4);
x3 = intVar(5);
y1 = intVar(0..100);
y2 = intVar(0..100);
y3 = intVar(0..100);
abstraction([y1, y2, y3], [x1, x2, x3], [4, 8, 12, 16], 7);

In this example y1 = 7, y2 = 4 and y3 = 7.

Requirements

  • x and y must be arrays of the same size.
  • abstractValue must not be in the domain of any x[i].
