The KPIs section

Key Performance Indicators (KPIs) are expressions whose value somehow characterize a solution and its quality. They differ from objective expressions: objective expressions are passed to minimize or maximize and CP Optimizer actively works on improving their value. KPIs are not actively improved in that sense: achieving good values of KPIs is usually a side product of achieving good objective value. For example, objective could be to minimize total duriation of a schedule and KPI could be the total idle time of a bottleneck machine.

Every time CP Optimizer finds a solution it prints its KPI values in the log just after the objective value. For this reason all KPIs must have unique names. Evaluation of a KPI can also fail (e.g. division by zero) then its value is NaN.

In cpo file format KPIs are specified in section KPIs that simply lists names of the existing expressions or defines new ones.

Example


duration = ...
cost = ...
sumTaskLengths = ...

// Objective:
minimize(5*duration + 3*cost);

KPIs {
  duration;
  cost;
  idleTime = duration - sumTaskLengths;
}

The code above specifies three KPIs: duration, cost and idleTime where duration and cost already existed before (but they were not KPIs) and idleTime is a new expression.

Syntax

Any expression defined in section KPIs could be also used outside the section. KPIs could be only numeric expressions (types floatExpr, intExpr or boolExpr). If a KPI is specified using an alias (see aliasing) then alias name is printed in the log.

Multiple KPIs sections are allowed, KPIs from multiple sections are simply combined together.
