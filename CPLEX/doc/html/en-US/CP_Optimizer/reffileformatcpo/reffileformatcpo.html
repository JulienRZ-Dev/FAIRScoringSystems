CP Optimizer File Format Reference Manual

This is the CP Optimizer File Format Reference Manual.

  • CP Optimizer file format syntax
    This section describes the elements of the CPO file format such as statements, expressions, types, variables, constants or directives, and how to use them.
  • Functions
    The following functions can be used in a CP Optimizer model file (CPO file):
  • The search section
    A CPO file may contain an optional search section which specifies a variable fixing strategy that CP Optimizer should use while searching for solutions. The strategy is described by search phases.
  • The startingPoint section
    The optional section startingPoint specifies a (possibly partial) solution that could be used by CP Optimizer to start the search.
  • The KPIs section
    Key Performance Indicators (KPIs) are expressions whose value somehow characterize a solution and its quality. They differ from objective expressions: objective expressions are passed to minimize or maximize and CP Optimizer actively works on improving their value. KPIs are not actively improved in that sense: achieving good values of KPIs is usually a side product of achieving good objective value. For example, objective could be to minimize total duriation of a schedule and KPI could be the total idle time of a bottleneck machine.
  • The parameters section
    The optional section parameters specifies the CP Optimizer parameters that should be used for solving the model.
  • The internals section
    The section internals contains internal data of CP Optimizer produced by the model dump function: IloCP::dumpModel in the C++ API, IloCP.dumpModel in the Java™ API, CP.DumpModel in C# API or oplrun -d for OPL models.
