C#/.NET users

Provides details of new features and changes for C#/.NET users.

Deprecated CP.AlwaysEqual() methods

The following CP.AlwaysEqual() methods have been deprecated in CP Optimizer 20.1.0 and will be removed in a future release: 

  • 
public IConstraint AlwaysEqual(
	ICumulFunctionExpr f,
	int start,
	int end,
	int v
)
  • 
public IConstraint AlwaysEqual(
	ICumulFunctionExpr f,
	IIntervalVar a,
	int v
)

Instead use the following CP.AlwaysIn() methods: 

  • 
public IConstraint AlwaysIn(
	ICumulFunctionExpr f,
	int start,
	int end,
	int vmin,
	int vmax
)
  • 
public IConstraint AlwaysIn(
	ICumulFunctionExpr f,
	IIntervalVar a,
	int vmin,
	int vmax
)

Deprecated CP.Synchronize() methods

The following CP.Synchronize() methods have been deprecated in CP Optimizer 20.1.0 and will be removed in a future release:

  • 
public ISynchronize Synchronize(
	IIntervalVar a,
	IIntervalVar[] bs
)
  • 
public ISynchronize Synchronize(
	IIntervalVar a,
	IIntervalVar[] bs,
	string name
)

Instead, use a set of CP.StartAtStart() and CP.EndAtEnd() constraints between the master interval variables and the other interval variables. If the CP.Synchronize() constraint is meant to formulate the selection of a subset of intervals, consider using the CP.Alternative() constraint instead (with unit, multiple or variable cardinality).
