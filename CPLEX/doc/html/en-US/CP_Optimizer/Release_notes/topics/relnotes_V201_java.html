Java users

Provides details of new features and changes for Java users, as well as some limitations.

Deprecated IloCP.alwaysEqual() methods

The following IloCP.alwaysEqual() methods have been deprecated in CP Optimizer 20.1.0 and will be removed in a future release: 

  • 
public IloConstraint alwaysEqual(IloCumulFunctionExpr f,
                        int start,
                        int end,
                        int v)
                          throws IloException
  • 
public IloConstraint alwaysEqual(IloCumulFunctionExpr f,
                        IloIntervalVar a,
                        int v)
                          throws IloException

Instead use the following IloCP.alwaysIn() methods: 

  • public IloConstraint alwaysIn(IloCumulFunctionExpr f,
                     int start,
                     int end,
                     int vmin,
                     int vmax)
                       throws IloException
  • public IloConstraint alwaysIn(IloCumulFunctionExpr f,
                     IloIntervalVar a,
                     int vmin,
                     int vmax)
                       throws IloException

Deprecated IloCP.synchronize() methods

The following IloCP.synchronize() methods have been deprecated in CP Optimizer 20.1.0 and will be removed in a future release:

  • 
public IloSynchronize synchronize(IloIntervalVar a,
                         IloIntervalVar[] bs,
                         java.lang.String name)
                           throws IloException
  • 
public IloSynchronize synchronize(IloIntervalVar a,
                         IloIntervalVar[] bs)
                           throws IloException

Instead, use a set of IloCP.startAtStart() and IloCP.endAtEnd() constraints between the master interval variables and the other interval variables. If the IloCP.synchronize() constraint is meant to formulate the selection of a subset of intervals, consider using the IloCP.alternative() constraint instead (with unit, multiple or variable cardinality).
