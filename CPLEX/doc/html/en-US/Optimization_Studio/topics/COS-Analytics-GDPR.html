GDPR

General Data Protection Regulation (GDPR) has been adopted by the European Union ("EU") and applies from May 25, 2018.

Why is GDPR important?

GDPR establishes a stronger data protection regulatory framework for processing of personal data of individuals. GDPR brings:

  • New and enhanced rights for individuals
  • Widened definition of personal data
  • New obligations for processors
  • Potential for significant financial penalties for non-compliance
  • Compulsory data breach notification

Product Configuration - considerations for GDPR Readiness

Data handling in CPLEX Optimization Studio

In IBM ILOG CPLEX Optimization Studio, data is present in the following:

  • Data used with models to represent optimization problems. Typically, this data does not contain personal information. The data is used by CPLEX Optimization Studio for the purposes of finding solutions to the optimization problem. The data is accessed using a database connector or from files. It is not stored by CPLEX Optimization Studio.

  • Data written to log files by CPLEX Optimization Studio. These files do not contain personal information.

  • Username and password data if a user connects CPLEX Optimization Studio either to the IBM Watson Machine Learning service or to IBM ILOG CPLEX Enterprise Server. This username and password data can be entered and deleted using the IDE.

In IBM ILOG CPLEX Enterprise Server, data is present in the following:

  • Data submitted with models to represent optimization problems, as for CPLEX Optimization Studio.

  • Username and activity data for users who connect to the server. Management of this data is fully delegated to WebSphere Application Server or WebSphere Liberty.

The following sections provide considerations for configuring IBM ILOG CPLEX Optimization Studio to help your organization with GDPR readiness.

Configuration to support data handling requirements

The GDPR legislation requires that personal data is strictly controlled and that the integrity of the data is maintained. This requires the data to be secured against loss through system failure and also through unauthorized access or via theft of computer equipment or storage media.

If your use of IBM ILOG CPLEX Optimization Studio includes the use of personal data for your optimization problems, you need to ensure that data your data is secured against loss or unauthorized access.

Data Lifecycle

The data life cycle for IBM ILOG CPLEX Optimization Studio is

  • The customer creates an optimization model for use with data either from a database or a file.

  • The customer tests the model with the data and debugs the model.

  • Once the model performs successfully, it is deployed outside the studio.

The data life cycle for IBM ILOG CPLEX Enterprise Server is

  • The customer creates user accounts for CPLEX Enterprise Server using a WebSphere administration console.

  • Users submit jobs to CPLEX Enterprise Server. Data management is fully delegated to WebSphere.

Personal data used for online contact with IBM

IBM ILOG CPLEX Optimization Studio clients can submit online comments/feedback/requests to contact IBM about IBM ILOG CPLEX Optimization Studio subjects in a variety of ways, primarily:

  • Public comments area on pages in the IBM ILOG CPLEX Optimization Studio community on IBM developerWorks
  • Public comments area on pages of IBM ILOG CPLEX Optimization Studio documentation in IBM Knowledge Center
  • Public comments in the IBM ILOG CPLEX Optimization Studio space of dWAnswers
  • Feedback forms in the IBM ILOG CPLEX Optimization Studio community

Typically, only the client name and email address are used, to enable personal replies for the subject of the contact, and the use of personal data conforms to the IBM Online Privacy Statement.

Data Collection

Types of Data Collected

IBM ILOG CPLEX Optimization Studio may collect user credentials if a user chooses to connect the studio to either the IBM Watson Machine Learning service or IBM ILOG CPLEX Enterprise Server.

IBM ILOG CPLEX Enterprise Server uses usernames and passwords, managed by a WebSphere server, to associate jobs with different users. For more information on this data, consult the documentation for the WebSphere server you have deployed.

Data Storage

IBM ILOG CPLEX Optimization Studio may store user credentials if a user chooses to connect the studio to either the IBM Watson Machine Learning service or IBM ILOG CPLEX Enterprise Server. These credentials can be entered and deleted using the studio IDE.

IBM ILOG CPLEX Enterprise Server does not store user credentials. It delegates user management to the WebSphere application server.

Data Access

IBM ILOG CPLEX Optimization Studio is a single-user product. Access to user credentials is restricted to the currently active user.

IBM ILOG CPLEX Enterprise Server is a multi-user product. Data access is managed by the WebSphere application server that hosts CPLEX Enterprise Server.

Data Processing

IBM ILOG CPLEX Optimization Studio does not encrypt data that it processes as part of an optimization problem. The user credentials stored for access to the IBM Watson Machine Learning web service are encrypted. The user credentials stored for access to IBM ILOG CPLEX Enterprise Server are not encrypted.

Data Deletion

Right to Erasure

Article 17 of the GDPR states that data subjects have the right to have their personal data removed from the systems of controllers and processors - without undue delay - under a set of circumstances.

Data Deletion characteristics

If your use of IBM ILOG CPLEX Optimization Studio includes the use of personal data, you should implement appropriate controls and tools to satisfy the right to erasure.

IBM ILOG CPLEX Optimization Studio does not require any special method for data deletion. You are responsible for implementing appropriate methods for storage media to securely delete data, which includes media zeroization if necessary. You are also responsible for deleting data.

Data Monitoring

You should regularly test, assess, and evaluate the effectiveness of your technical and organizational measures to comply with GDPR. These measures should include ongoing privacy assessments, threat modeling, centralized security logging and monitoring among others.

Responding to Data Subject Rights

The personal data stored or processed by IBM ILOG CPLEX Optimization Studio falls under the following categories:

  • Basic user data, such as usernames and passwords
  • Personal data that could be used as an input to an optimization problem.

If you use personal data as an input to optimization problems, you should consider and implement methods so that you can respond to a request to:

  • Delete data
  • Correct data
  • Modify data
  • Extract specific data for export to another system
  • Restrict the use of the data within the overall system

securely and responsively.
