Cluster edition entitlements

The cluster edition is a deployment metric for CPLEX Optimization Studio. It consists of double entitlements per "establishment" and per "process". The user must have entitlement to at least 1 unit of each to be able to use it.

For each entitlement the standard License Information document gives a definition what it covers exactly here License Information documents.

How to relate the "process" metric to technical measurement can be established as follows:

  • In the CPLEX engine the user can set the thread parameter for each solve which determines how much resources the solve will require from the system (hardware/OS/VM). X process entitlement allows the user to set the thread parameter to X.

  • In the CP Optimizer engine the user can set the so-called worker parameter for each solve which determines how much resources the solve will require from the system (hardware/OS/VM). X process entitlement allows the user to set the worker parameter to X. The number of thread/worker entitlements can be distributed among solves, i.e. having X number of threads/workers allows the user to run one solve with X threads/workers or X solves with 1 thread/worker each or anything in between as long as the sum of the total number of threads/workers does not exceed X.

    This distribution of threads/workers can happen among different establishment entitlements, i.e. the sum of the total amount of threads/workers across all the separate clusters covered by separate establishment entitlements can not exceed the purchased entitlement of threads/workers.
