Other new features in OPL

These release notes highlight other changes in 20.1.0 to the Optimization Programming Language (OPL), including changes to the OPL interfaces and the IBM ILOG Script for OPL.

New constraint strictLex

A new constraint strictLex, which states that the first array of variables is strictly less than the second array of variables in the lexicographic order, has been added to OPL. See strictLex.

New methods in IBM ILOG Script for OPL

The following new methods have been added:

  • IloOplConflictIterator.recalculate() - This method clears any cached conflict and forces recomputation of the conflict. Note that conflicts are cached. If you modify the model for which a conflict was computed then the cached conflict is not deleted. In order to delete a cached conflict you should call recalculate.
  • IloOplRelaxationIterator.recalculate() - This method clears any cached relaxations and forces recomputation of the relaxation. Note that relaxations are cached. If you modify the model for which a relaxation was computed then the cached relaxation is not deleted. In order to delete a cached relaxation you should call recalculate.

Defining the IL_STD macro when compiling Concert C++ applications no longer required

Previous versions of the OPL C++ interface required you to compile Concert C++ applications with -DIL_STD. This is no longer required.
