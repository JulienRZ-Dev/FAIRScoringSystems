or

OPL aggregate for alternative constraints.

Purpose

OPL aggregate constraint to express alternative constraints more compactly.

context type
Model files (.mod) - Not allowed in a CPLEX constraint block. 
boolean (1 if the constraint is true, 0 otherwise)

Syntax

AggregateExpression: "sum" '(' Qualifiers ')' Expression
                   | "min" '(' Qualifiers ')' Expression
                   | "max" '(' Qualifiers ')' Expression
                   | "prod" '(' Qualifiers ')' Expression
                   | "or" '(' Qualifiers ')' Expression
                   | "and" '(' Qualifiers ')' Expression
                   | "union" '(' Qualifiers ')' Expression
                   | "inter" '(' Qualifiers ')' Expression
                   | AllExpression

Description

This constraint enables you to express alternative constraints in a more compact form by using a logical OR to aggregate several constraints into a single one. Behavior on empty sets:

or(empty set)=false

Example

using CP;

int n = 5;

range R = 1..n;
dvar int x[R] in R;

maximize sum(i in R)x[i];

subject to {
  or(i in 1..5:i % 2==1) x[i]==1;
  or(i in 1..5:i % 2==0) x[i]==2;
}
