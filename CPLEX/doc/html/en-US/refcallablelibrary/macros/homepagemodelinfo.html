Modeling information in the CPLEX Callable Library (C API)

These symbols identify modeling information available from CPLEX when your application invokes data checking routines. Typically, CPLEX identifies this modeling information during a solve after you have set the data consistency checking and modeling assistance parameter to the value 2; that is, the information may not yet be available as you read your model. The names of these symbols signify the kind of information available:
  • CPX specifies the namespace or scope CPLEX.
  • MI specifies modeling information.
  • BIGM refers to a Big M formulation (likely to create trickle flow or other unstable situations).
  • COEF specifies coefficients.
  • EPGAP specifies an epsilon gap; that is, a very small amount.
  • IND specifies indicator constraints.
  • INEQ specifies an inequality.
  • KAPPA refers to kappa, also known as the condition number, a statistical evaluation of bases or models.
  • LARGE_NUM specifies a large number.
  • LB specifies lower bound.
  • LC specifies lazy constraints.
  • LINNZ specifies linear nonzeros.
  • NZ specifies nonzero (such as nonzero entries in the problem matrix).
  • OBJ specifies an objective function.
  • OBJOFFSET specifies an offset from the objective function.
  • QC specifies a quadratic constraint; that is, a constraint in which one or more terms are quadratic.
  • QNZ specifies quadratic nonzeros.
  • QOBJ specifies a quadratic term or terms in the objective function.
  • RHS specifies righthand side.
  • SMALL_NUM specifies a small number.
  • UB specifies upper bound.
  • UC specifies user cuts.
  • VAR specifies variables.
