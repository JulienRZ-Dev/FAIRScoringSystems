CPXXgetsolnpoolsolnname and CPXgetsolnpoolsolnname

The routine CPXXgetsolnpoolsolnname/CPXgetsolnpoolsolnname accesses the name of a solution, specified by the argument soln, of the solution pool associated with the problem object specified by the argument lp.

int  CPXXgetsolnpoolsolnname( CPXCENVptr env, CPXCLPptr lp, char * store, CPXSIZE storesz, CPXSIZE * surplus_p, int which )

int  CPXgetsolnpoolsolnname( CPXCENVptr env, CPXCLPptr lp, char * store, int storesz, int * surplus_p, int which )

Description

The routine CPXXgetsolnpoolsolnname/CPXgetsolnpoolsolnname accesses the name of a solution, specified by the argument soln, of the solution pool associated with the problem object specified by the argument lp.

Note: If the routine returns CPXERR_NEGATIVE_SURPLUS, then the value in *surplus_p is negative and specifies the difference between the memory provided and the memory required (in bytes).

A nonnegative value of surplus_p specifies that the length of the array store was sufficient. A negative value specifies that the length of the array was insufficient and that the routine could not complete its task. In this case, CPXXgetsolnpoolsolnname/CPXgetsolnpoolsolnname returns the value CPXERR_NEGATIVE_SURPLUS, and the negative value of the variable surplus_p specifies an upper limit on the amount of insufficient space in the array store.

Arguments

env
A pointer to the CPLEX environment as returned by CPXXopenCPLEX/CPXopenCPLEX.
lp
A pointer to a CPLEX problem object as returned by CPXXcreateprob/CPXcreateprob.
store
A pointer to a buffer of size storesz. It may be NULL if storesz is 0 (zero).
storesz
An integer specifying the length of the array store. It may be 0 (zero).
surplus_p
A pointer to an integer to contain an upper limit on the difference between storesz and the amount of memory required to store the name of the solution.
which
An integer specifying the index of the solution for which the name is returned.

Return

The routine returns 0 (zero) if successful and nonzero if an error occurs. The value CPXERR_NEGATIVE_SURPLUS specifies that insufficient space was available in the array store to hold the name of the solution.

Example


 CPXgetsolnpoolsolnname (env, lp, namei, BUFSIZE, &surplus, i);
