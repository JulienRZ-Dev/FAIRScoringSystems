CPXXkillpnorms and CPXkillpnorms

The routine CPXXkillpnorms/CPXkillpnorms deletes any primal steepest-edge norms that have been retained relative to an active basis.

int  CPXXkillpnorms( CPXLPptr lp )

int  CPXkillpnorms( CPXLPptr lp )

Description

Warning:

This is an advanced routine. Advanced routines typically demand a thorough understanding of the algorithms used by CPLEX. Thus they incur a higher risk of incorrect behavior in your application, behavior that can be difficult to debug. Therefore, the team encourages you to consider carefully whether you can accomplish the same task by means of other Callable Library routines instead.

The routine CPXXkillpnorms/CPXkillpnorms deletes any primal steepest-edge norms that have been retained relative to an active basis. If the user believes that the values of these norms may be significantly in error, and the setting of the parameter that specifies the primal simplex pricing algorithm (CPXPARAM_Simplex_PGradient) is CPX_PPRIIND_STEEP, calling CPXXkillpnorms/CPXkillpnorms means that fresh primal steepest-edge norms will be computed on the next call to CPXXprimopt and CPXprimopt .

Arguments

lp
A pointer to a CPLEX LP problem object, as returned by CPXXcreateprob/CPXcreateprob.

Return

The routine returns 0 (zero) if successful or an error code otherwise.
