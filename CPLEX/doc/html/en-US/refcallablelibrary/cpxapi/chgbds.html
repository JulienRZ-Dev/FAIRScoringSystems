CPXXchgbds and CPXchgbds

The routine CPXXchgbds/CPXchgbds changes the lower or upper bounds on a set of variables of a problem.

int  CPXXchgbds( CPXCENVptr env, CPXLPptr lp, CPXDIM cnt, CPXDIM const * indices, char const * lu, double const * bd )

int  CPXchgbds( CPXCENVptr env, CPXLPptr lp, int cnt, int const * indices, char const * lu, double const * bd )

Description

The routine CPXXchgbds/CPXchgbds changes the lower or upper bounds on a set of variables of a problem. Several bounds can be changed at once, with each bound specified by the index of the variable with which it is associated. The value of a variable can be fixed at one value by setting the upper and lower bounds to the same value.

Unbounded Variables

If a variable lacks a lower bound, then CPXXgetlb/CPXgetlb returns a value less than or equal to -CPX_INFBOUND.

If a variable lacks an upper bound, then CPXXgetub/CPXgetub returns a value greater than or equal to CPX_INFBOUND.

These conventions about unbounded variables should be taken into account when you change bounds with CPXXchgbds/CPXchgbds.

CPLEX treats any lower bound that is set to a value less than or equal to that of the constant -CPX_INFBOUND as negative infinity. Likewise, CPLEX treats any upper bound that is set to a value greater than or equal to that of the constant CPX_INFBOUND as infinity. The constant CPX_INFBOUND is declared in the header file cplexx.h/cplex.h.

Arguments

env
A pointer to the CPLEX environment as returned by CPXXopenCPLEX/CPXopenCPLEX.
lp
A pointer to a CPLEX problem object as returned by CPXXcreateprob/CPXcreateprob.
cnt
An integer that specifies the total number of bounds to be changed, and thus specifies the length of the arrays indices, lu, and bd.
indices
An array of length cnt containing the numeric indices of the columns corresponding to the variables for which bounds are to be changed. This argument can be NULL if cnt is 0 (zero).
lu
An array of length cnt containing characters that tell whether the corresponding entry in the array bd specifies the lower or upper bound on column indices[j]. This argument can be NULL if cnt is 0 (zero). Possible values appear in the table.
bd
An array of length cnt containing the new values of the lower or upper bounds of the variables present in indices. This argument can be NULL if cnt is 0 (zero).

Return

The routine returns 0 (zero) if successful and nonzero if an error occurs.

Example


status = CPXchgbds (env, lp, cnt, indices, lu, bd);
Table 1. Values of lu denoting lower or upper bound in indices[j]
lu[j] = 'L' bd[j] is a lower bound
lu[j] = 'U' bd[j] is an upper bound
lu[j] = 'B' bd[j] is the lower and upper bound
