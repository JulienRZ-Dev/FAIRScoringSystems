Data checking in the CPLEX Callable Library (C API) Reference Manual

These routines offer you a means of checking data in your models as part of making sure that your application solves the problem that you want to solve. For more information about this idea of data checking, see the topic Test data in the CPLEX User's Manual.

  • CPXXcheckaddcols and CPXcheckaddcols
    Checks data when columns are added to a model.
  • CPXXcheckaddqconstr and CPXcheckaddqconstr
    Checks data when a constraint containing a quadratic term is added to a model.
  • CPXXcheckaddrows and CPXcheckaddrows
    Checks data when rows are added to a model.
  • CPXXcheckchgcoeflist and CPXcheckchgcoeflist
    Checks data when the list of coefficients in a model changes.
  • CPXXcheckcopyctype and CPXcheckcopyctype
    Checks the data about the type of variables copied in a model.
  • CPXXcheckcopylp and CPXcheckcopylp
    Checks the data copied by CPXXcopylp/CPXcopylp.
  • CPXXcheckcopylpwnames and CPXcheckcopylpwnames
    Checks data copied by CPXXcopylpwnames/CPXcopylpwnames.
  • CPXXNETcheckcopynet and CPXNETcheckcopynet
    Checks data copied to a network object.
  • CPXXcheckcopyqpsep and CPXcheckcopyqpsep
    Checks the data when the quadratic objective matrix Q for a separable QP problem is copied.
  • CPXXcheckcopyquad and CPXcheckcopyquad
    Checks data when a quadratic objective matrix Q is not diagonal.
  • CPXXcheckcopysos and CPXcheckcopysos
    Checks data copied for a special ordered set (SOS) in a problem object of type MILP, MIQP, or MIQCP.
  • CPXXcheckvals and CPXcheckvals
    Checks data by comparing an array of indices and an array of values for errors.
