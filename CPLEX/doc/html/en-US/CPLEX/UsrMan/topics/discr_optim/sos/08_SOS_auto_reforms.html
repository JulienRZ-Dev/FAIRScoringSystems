Automatic reformulation of special ordered sets

Describes the automatic reformulation of special ordered sets available in CPLEX.

A special ordered set (SOS) can be used for modeling complex nonlinearities, for example, disjunctive constraints or piecewise linear functions, in an easy and natural way. However, in order to achieve good performance and effectively solve the problem, during the solution process it can be advantageous to reformulate the special ordered sets as linear constraints, by introducing additional variables into the problem formulation.

CPLEX implements a logarithmic reformulation of special ordered sets, where each SOS is reformulated by adding to the problem a number of binary variables and extra linear constraints which are logarithmic in the size of the SOS. By default, CPLEX automatically decides whether a SOS should be reformulated or not. However, the user can explicitly control the reformulation of special ordered sets of type 1 (SOS1) and type 2 (SOS2) through, respectively, the SOS1 reformulations (CPXPARAM_Preprocessing_SOS1Reform) parameter and the SOS2 reformulations (CPXPARAM_Preprocessing_SOS2Reform) parameter.

For a more thorough exploration of the theory supporting SOS reformulations, see, for example, the following research: J.P. Vielma and G.L. Nemhauser, "Modeling disjunctive constraints with a logarithmic number of binary variables and constraints," in Mathematical Programming, volume 128, pages 49-72, 2011.

Note:

The reformulation of special ordered sets of type 2 (SOS2) implicitly affects the performance of the solver also on problems formulated with piecewise linear (PWL) functions, since PWL functions in the problem are internally reformulated with SOS2. See Using piecewise linear functions in optimization: a transport example.
