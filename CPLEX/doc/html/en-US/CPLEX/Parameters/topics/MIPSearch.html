Description

Sets the search strategy for a mixed integer program (MIP). By default, CPLEX chooses whether to apply dynamic search or conventional branch-and-cut based on characteristics of the model and the presence (or absence) of callbacks.

Only the generic callbacks and informational callbacks are compatible with dynamic search.

For more detail about generic callbacks and how to create and install them in your application, see Generic callbacks in the CPLEX User’s Manual

For more detail about informational callbacks and how to create and install them in your application, see Informational callbacks in the CPLEX User’s Manual.

To benefit from dynamic search, a MIP must not include legacy query callbacks. In other words, legacy query callbacks are not compatible with dynamic search. For a more detailed definition of query or diagnostic callbacks, see Query or diagnostic callbacks in the CPLEX User’s Manual.

To benefit from dynamic search, a MIP must not include legacy control callbacks. In other words, control callbacks are not compatible with dynamic search. For a more detailed definition of legacy control callbacks, see Advanced MIP control interface (legacy) in the CPLEX User’s Manual. If control callbacks are present in your application, CPLEX will disable dynamic search. To control the MIP algorithm and still use dynamic search one can use the new generic callback framework cited above.

Table 1. Values
Value Symbolic Name Meaning
0 CPX_MIPSEARCH_AUTO Automatic: let CPLEX choose; default
1 CPX_MIPSEARCH_TRADITIONAL Apply traditional branch and cut strategy; disable dynamic search
2 CPX_MIPSEARCH_DYNAMIC Apply dynamic search
