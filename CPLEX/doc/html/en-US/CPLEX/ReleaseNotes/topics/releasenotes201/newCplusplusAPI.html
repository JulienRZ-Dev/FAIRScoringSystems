New features in the C++ API

These release notes highlight new features in the C++ API of IBM CPLEX 20.1.0. For more information, see the reference manual of the C++ API of CPLEX.

Defining the IL_STD macro when compiling Concert C++ applications no longer required

Previous versions of CPLEX required you to compile Concert C++ applications with -DIL_STD. This is no longer required.

New conflict refiner example

The following new example, demonstrating how to use the conflict refiner, has been added in CPLEX 20.1.0:

  • <Install_dir>/cplex/examples/cpp/iloconflictex1.cpp

New enumeration field to report number of open nodes from generic callback

This enum identifies the kinds of information that your application can retrieve from a generic callback by means of the methods IloCplex::Callback::getIntInfo(Info), IloCplex::Callback::getLongInfo(Info), or IloCplex::Callback::getDoubleInfo(Info). It contains one new field in CPLEX 20.1:

  • NodesLeft = CPXCALLBACKINFO_NODESLEFT

Please see the documentation of CPXCALLBACKINFO for more information.

New member function IloCplex::readStartInfo()

The new member function IloCplex::readStartInfo() has been added. This member function reads starting information from the SOL file denoted by name and copies this information into the invoking object.
