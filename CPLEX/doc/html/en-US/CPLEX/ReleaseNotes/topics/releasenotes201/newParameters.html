New parameters in CPLEX

These release notes highlight new parameters in IBM CPLEX 20.1.0

New choice added to MIP emphasis switch parameter

A new choice has been added to the parameter MIP emphasis switch, CPXPARAM_Emphasis_MIP. (Please note that, prior to CPLEX V12.6.0, this parameter was named CPX_PARAM_MIPEMPHASIS.)

This new choice, 5 (MIPEmphasisHeuristic in Concert Technology or, in the Callable Library, CPX_MIPEMPHASIS_HEURISTIC), works exclusively at finding high quality feasible solutions as early as possible. A best bound is also computed, but almost no effort is put into proving optimality. This setting works even harder than choice 4, (CPX_MIPEMPHASIS_HIDDENFEAS), on finding high quality solutions, and relies on a very aggressive usage of many primal heuristics. Therefore, it can be useful for cases where even choice 4 has difficulty finding solutions of acceptable quality within the required computing time.

For more information, see Emphasizing feasibility and optimality.

New parameter to set presolve reformulations

The new parameter reformulations, CPXPARAM_Preprocessing_Reformulations, allows you to set presolve reformulations.

New parameter node cuts switch

The new parameter node cuts switch, CPXPARAM_MIP_Cuts_Nodecuts, allows you to decide whether or not cutting planes are separated at the nodes of the branch-and-bound tree.

New parameters for special ordered sets

CPLEX 20.1.0 implements an automatic logarithmic reformulation of special ordered sets of type 1 (SOS1) and type 2 (SOS2):

  • SOS1 reformulations, CPXPARAM_Preprocessing_SOS1Reform
  • SOS2 reformulations, CPXPARAM_Preprocessing_SOS2Reform

For more information, see Automatic reformulation of special ordered sets in the CPLEX User's Manual.
