Current callback configuration now displayed before optimization

CPLEX 20.1.0 now allows for the display of the current callback configuration before solving.

The output is controlled by changed parameter display switch, CPXPARAM_ParamDisplay, and the display can be turned on and off using this parameter.

Generic callbacks

For the generic callback, if a non-zero contextmask is set, then that is displayed.

Here is an example of output for a generic callback, with a non-zero contextmask set:

Generic callback                                 0x358

Here is a list of the possible contexts: Callback context.

Legacy callbacks

For legacy callbacks, a combination of characters is displayed, each character representing a different callback that has been set.

Here is an example of output for a legacy callback, using the character list:

Legacy callback                                  ctXBLUHSN

The map of characters for the legacy callback display is:

  • c: LP callback
  • p: profiling callback
  • t: tuning callback
  • i: MIP info callback
  • X: MIP callback
  • B: branch callback
  • L: lazy constraint callback
  • U: user cut callback
  • H: heuristic callback
  • I: incumbent callback
  • S: solve callback
  • D: delete node callback
  • N: node callback
