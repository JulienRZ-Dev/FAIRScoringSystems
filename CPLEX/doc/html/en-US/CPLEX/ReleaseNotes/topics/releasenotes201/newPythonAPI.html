New features in the CPLEX Python API

These release notes highlight new features in the Python API of IBM CPLEX 20.1.0. For more information, see the reference manual of the Python API of CPLEX.

Python versions

CPLEX 20.1.0 supports these versions of Python on platforms that previously supported CPLEX and Python:

  • Python V3.7
  • Python V3.8

New enumeration values for generic callback information

CPLEX 20.1.0 has two new enumeration values for the generic callback information enumeration:

  • after_cut_loop - New enumeration value CPXCALLBACKINFO_AFTERCUTLOOP that reports whether CPLEX has finished its internal cut loop. The information can only be queried from the RELAXATION context.
  • nodes_left - New enumeration value CPXCALLBACKINFO_NODESLEFT that reports the number of open nodes from generic callback. The information can only be queried from the RELAXATION context.

Please see the documentation of CPXCALLBACKINFO for more information.
